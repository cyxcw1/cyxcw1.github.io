解决SAE上的terrifico主题评论不正常

发表于 2014-03-09 | 更新于 2019-01-13 | 分类于 本站建设

现象1：点击reply，只看到滚动条在动，无法正常显示评论框体。

解决：Chrome审查元素–>Console模块，观察到点击reply时，提示addComment没有定义。在element里看到reply绑定的click函数为addComment。搜索文件，该函数已经在javascript文件comment-reply.js文件里定义，并在framework-functions.php文件里面导入该JS：

if ( is_singular() && comments_open() && get_option( ‘thread_comments’ ) ){
    wp_enqueue_script( ‘comment-reply’ );
}

应该是这一堆判断条件没有通过，导致JS文件无法导入。网上也有人反映is_singual()函数判断有bug。于是删除了这些判断条件，直接导入JS文件：

wp_enqueue_script( ‘comment-reply’ );

把修改后的文件上传到服务器，一切正常：

 

 

现象2：在有评论（评论数量不为0）的文章里，无法显示评论框体。

解决：查看comment.php文件，已经有了显示评论窗体的代码：

<?php comment_form(); ?>

胡乱调试下，发现加个echo能把问题解决：

<?php
    echo “Hi man”;
    comment_form();
?>

虽然看起来有点囧，为嘛上面会蹦出来一个Hi man?（本来想echo “n”，或者一个笑脸”:)”，很不幸不能解决问题）
